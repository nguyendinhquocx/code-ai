Dựa vào TOÀN BỘ cuộc hội thoại trước đó và ngữ cảnh hiện tại, tạo cho tao BỘ TÀI LIỆU PHÁT TRIỂN HOÀN CHỈNH với 13-15 documents.

Mày phải TỰ ĐỘNG TRÍCH XUẤT:
- Thông tin dự án (tên, loại, mục tiêu)
- Yêu cầu từ Mini-PRD
- Quyết định thiết kế
- Stack công nghệ
- Tất cả chi tiết đã thảo luận

Tạo ĐẦY ĐỦ 15 DOCUMENTS sau (mỗi cái trong 1 code block riêng):

1. Mini-PRD hoàn chỉnh
2. User Context chi tiết
3. Constraints List đầy đủ
4. Tech Stack Decision (có so sánh alternatives)
5. Wireframes UI (ASCII art cho TẤT CẢ màn hình - mobile + desktop)
6. Component Architecture (sơ đồ cây đầy đủ)
7. Design System (colors, typography, spacing, breakpoints chi tiết)
8. User Flows (tất cả luồng chính và phụ)
9. System Architecture (sơ đồ kiến trúc đầy đủ)
10. API Endpoints Specification (nếu cần)
11. Development Roadmap (chia theo ngày/giờ với checklist chi tiết)
12. Performance Budget (Lighthouse targets cụ thể)
13. Security Checklist (đầy đủ)
14. File Structure (cấu trúc thư mục project hoàn chỉnh)
15. Progress Tracking Template (để AI agent track tiến độ)

FORMAT YÊU CẦU:
- Mỗi document trong code block markdown riêng
- Tiêu đề rõ ràng với số thứ tự
- Nội dung HOÀN CHỈNH, KHÔNG viết tắt, KHÔNG "..."
- Sẵn sàng copy-paste vào file
- Cuối cùng tạo thêm 1 code block cho DIRECTORY STRUCTURE đề xuất

BẮT ĐẦU TẠO NGAY!
